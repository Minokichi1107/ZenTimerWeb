⑭ JSONタイムスタンプからコード自動登録 — 確定仕様

次の行のタイムスタンプまでの間のコードを全部その行に並べる
タイムスタンプなし行は全行均等割りで時刻を自動計算して割り当て
既存コードがある行は上書き確認ダイアログ（行ごとではなく一括で「上書きしますか？」）

⑫ コードダイアグラム拡張 確定仕様
編集・削除対象： 既存バリアント（ロー・バレー等）もカスタム登録も両方編集・削除可能
UI： バリアントカードにホバーで右上に ✏️ 🗑 ボタンが出る

✏️ → 手動登録と同じ入力フォームが開いてフレット値を編集可能
🗑 → 確認なしで即削除（そのコードのバリアントが0になったらDBから除去）
変更はlocalStorageに自動保存（既存の永続化の仕組みを使う）

コメントアウト対象（訂正）：
右パネルのバリアントラベル（「ロー」「バレー3F」「カスタム」等のテキスト）ではなく、現状表示されていないため実際には不要です。もう少し具体的に教えてください — 現在右パネルで「邪魔だ・不要だ」と感じている表示はどれですか？ スクリーンショットか「〇〇という文字」と教えていただけると助かります。


⑪ プロジェクトライブラリ — 完全確定仕様
保存・管理：

初回起動時にライブラリフォルダを選択（File System Access API）
以降は自動でそのフォルダを参照、設定から変更可能
フォルダ内の .json を自動スキャンして一覧化
クラウドストレージはユーザーがフォルダを指定するだけで自動対応

UI：

ヘッダーに「📚 ライブラリ」ボタン → フルスクリーンモーダル
テーブル形式の一覧 + 検索・ソート・フィルター
設定ボタン（⚙️）でライブラリフォルダの変更が可能

表示項目：
曲名・アーティスト・Key・BPM・拍子・行数・最終保存日時
操作：
開く・削除・リネーム・複製
JSONへの追加フィールド：
json{
  "title": "曲名",
  "artist": "",
  "key": "Gm",
  "tempo": 126,
  "beats": 4,
  ...
}
左パネルへの追加：

アーティスト名入力欄をKey/BPMの近くに追加
拍子（beats）入力欄も追加（デフォルト4）
